1. INSTALLATION
--------------------------------------------------------------------------------

   a) Place the Montoya API JAR (montoya-api.jar) into the lib/ folder.
      - Get it from: https://github.com/PortSwigger/burp-extensions-montoya-api

   b) Build the extension:
      Open terminal in the project folder and run:

        mvn clean package -q

      Output JAR will be at: target/PentestChecklistGenerator.jar

   c) Load in Burp Suite Pro:
      - Go to Extensions tab
      - Click "Add"
      - Extension Type: Java
      - Select: target/PentestChecklistGenerator.jar
      - Click "Next"

   d) A new tab "RepeaterSync" will appear in Burp Suite.
      The console will show:

        RepeaterSync V1.337 — VulnDB-Powered
        VulnDB: 54 sub-vulns | 19 TOs | CWE mapping


2. CONFIGURATION (One-Time Setup)
--------------------------------------------------------------------------------

   Open the "RepeaterSync" tab in Burp Suite.

   Engine Mode:
     - Local Engine   — Instant, offline, zero API cost. Uses built-in
                         VulnDB with 54 sub-vulnerabilities for smart mapping.
     - OpenAI API     — Sends tab data to OpenAI GPT-4o-mini. Requires API key.
     - Claude API     — Sends tab data to Claude. Requires API key.

   AI Chat URL (optional):
     - Used for "Polish with AI Chat" and "Copy Prompt + Screenshot" features.
     - Default: https://chatgpt.com/
     - Change to https://claude.ai/ if you prefer Claude.

   Output Excel:
     - Set the path where the final checklist Excel will be saved.
     - Default: Desktop/Pentest_Checklist.xlsx
     - Click "..." to browse.

   Click "Save Config" after making changes.


3. ADDING TABS (4 Methods)
--------------------------------------------------------------------------------

   Method A — Auto-Capture (Recommended):
     - Just use Burp Repeater normally.
     - Every request you send in Repeater is automatically captured.
     - Tab names appear in the extension's tab list.
     - Tip: Add notes to Repeater tabs (right-click > Add Note) for
       better tab names. The extension uses your notes as the tab name.

   Method B — Right-Click Context Menu:
     - In Repeater, right-click any request.
     - Select "Send to Checklist Generator".
     - The tab with its request/response data will be added.

   Method C — Manual Entry:
     - Click "+ Manual Tab" button.
     - Type the tab name, e.g.: F - TO.01 - SQL Injection blind
     - Click OK.

   Method D — Paste Tab Names:
     - Click "Paste Tab Names (JSON/Text)".
     - Paste a JSON array: ["F - SQLi error", "NF - XSS reflected"]
     - Or paste plain text (one tab name per line).


4. TAB NAMING CONVENTION
--------------------------------------------------------------------------------

   The smart engine detects TO mapping, F/NF status, CWE, and sub-vulnerability
   from your tab name. Use any of these formats:

   Found Prefix Formats:
     F - SQL Injection blind
     [F] IDOR - user account
     Found - XSS
     f/XSS reflected
     SSRF - Blind DNS - F              (suffix also works)
     XXE - F                            (suffix also works)

   Not Found Prefix Formats:
     NF - CSRF
     [NF] Session Fixation
     Not Found - SSRF
     nf/Open Redirect

   With TO Code (highest confidence):
     F - TO.01 - SQL Injection error based
     NF - TO.04 - XSS reflected

   With CWE ID:
     VA07_CWE-79-TO.04 - Stored XSS
     CWE-352 - CSRF missing token

   With VA Prefix (auto-detected as Found):
     VA01_CWE-89-TO.01 - SQL Injection
     VA03_CWE-799-TO.06 - Lack of Anti-Automation

   Multiple Vulns in One Tab:
     VA03_CWE-799-TO.06 - Lack of Anti-Automation, VA04_CWE-693-TO.06 - Missing CSP

   Plain Descriptive (keyword matching):
     HTML Injection via Announcement Content Parameter
     SSRF - Blind DNS
     Brute Force login no lockout


5. GENERATING THE CHECKLIST
--------------------------------------------------------------------------------

   Step 1: Make sure your tabs are listed in the table.

   Step 2: Click "Step 2: Generate Checklist".

   What happens (Local Engine mode):
     - Each tab name is analyzed by the VulnDB smart engine.
     - TO code, sub-vulnerability, and CWE are auto-detected.
     - F/NF status determines "Vulnerabilities Identified" or not.
     - Professional comments are auto-generated.
     - Results appear in the results table instantly.

   What happens (API mode):
     - Each tab is sent to OpenAI/Claude API with request/response data.
     - AI returns TO mapping, status, result, and professional comment.
     - If API fails, falls back to Local Engine automatically.


6. REVIEWING RESULTS
--------------------------------------------------------------------------------

   The results table has 10 columns:

     Tab Name          — Original tab name from Repeater
     TO Code           — Auto-detected (editable)
     Sub-Vulnerability — e.g. "SQL Injection - Error Based" (editable)
     CWE               — e.g. "CWE-89" (editable)
     Status            — Covered / Not Applicable / Not Covered (dropdown)
     Result            — Vulnerabilities Identified / No Vuln Identified (dropdown)
     Conf%             — Confidence score (color-coded)
     Comment           — Auto-generated professional comment (editable)
     Source            — Local Engine / API / AI Polished
     Confirm           — Checkbox to confirm reviewed

   Color Coding:
     Green  (80%+)   — High confidence, likely correct
     Yellow (55-79%) — Medium confidence, review recommended
     Red    (<55%)   — Low confidence, manual review needed

   All fields are editable. Override anything the engine got wrong.


7. POLISHING WITH AI CHAT (Optional, Free)
--------------------------------------------------------------------------------

   After generating with Local Engine, you can improve comments for free:

   a) Click "Polish with AI Chat".
   b) Choose "All Rows" or "Low-Confidence Only".
   c) The prompt is copied to your clipboard automatically.
   d) Your AI chat (ChatGPT/Claude) opens in the browser.
   e) Paste the prompt (Ctrl+V) in the chat.
   f) Copy the JSON response from AI.
   g) Back in the extension, click "Paste AI Response".
   h) Comments are updated with polished, professional versions.


8. SCREENSHOT WORKFLOW (Tab Name Extraction)
--------------------------------------------------------------------------------

   If you want AI to read your Repeater tab names from a screenshot:

   a) Take a screenshot of your Repeater tab bar:
      - Press Win+Shift+S (Windows) and select the tab bar area.

   b) Click "Paste Screenshot (Ctrl+V)" in the extension.
      - Status will show "Pasted (WxH) - ready".

   c) Click "Copy Prompt + Screenshot":
      - The prompt is copied to clipboard.
      - The screenshot is saved as a temp PNG file.
      - File Explorer opens with the screenshot selected.
      - AI chat opens in your browser.

   d) In the AI chat:
      - Paste the prompt (Ctrl+V).
      - Drag the screenshot file into the chat.
      - AI will return a JSON array of tab names.

   e) Copy the JSON response.

   f) Back in the extension, click "Paste Tab Names (JSON/Text)".
      - Tab names are added to the table.


9. EXPORTING TO EXCEL
--------------------------------------------------------------------------------

   a) Review and confirm all results.

   b) Click "Step 4: Export Excel".
      - If there are unconfirmed low-confidence rows, you'll be warned.

   c) The Excel file is generated in exact bank v4 template format:
      - 19 TOs (TO.01 through TO.99)
      - Proper merged cells, dropdowns, color coding
      - Status column color coding:
          Green  = Covered
          Yellow = Not Applicable
          Orange = Not Covered
      - Results column: Plain text (no background color)

   d) Comment Format for Found Vulnerabilities:

        VAXX_CWE-89-TO.01 - SQL Injection - Error Based. Tested input
        parameters for SQL injection. The application returned database
        error messages when crafted SQL syntax was injected...

      The "VAXX" is a placeholder. Replace XX with your report VA number
      when writing the final report.

   e) Comment Format for Not Found:

        Tested for injection vulnerabilities including SQLi, command
        injection, template injection, host header injection, and
        related vectors. The application handles invalid input properly
        and no injection points were identified.

      No VAXX_ prefix is added for "No Vulnerability Identified" rows.

   f) Multiple test cases for the same TO are merged automatically:
      - If ANY finding for a TO is "Found", result = "Vulnerabilities Identified"
      - Comments from all findings are combined with " | " separator
      - Highest confidence score is kept

   g) The Excel file opens automatically after export.

11. TIPS & BEST PRACTICES
--------------------------------------------------------------------------------

   - Use descriptive tab names in Repeater for best detection accuracy.
       Good:  "F - TO.01 - SQLi error based on search param"
       Bad:   "test1"

   - Add Repeater notes (right-click tab > Rename) before sending requests.
     The extension uses your tab notes as the tab name.

   - For fastest workflow with zero cost:
       1. Use Local Engine mode
       2. Generate checklist
       3. Polish only low-confidence rows via AI Chat
     This gives you professional-grade comments without any API spend.

   - The VulnDB recognizes CWE IDs directly. Including CWE-89 in your
     tab name maps it to TO.01 automatically, even without other keywords.

   - Config is saved to ~/.pentest-checklist/config.properties
     and persists across Burp restarts.

   - The extension auto-captures every Repeater request. If you don't want
     a tab, select it in the table and click the X button to remove it.

   - You can edit any cell in the results table before exporting. The
     extension respects your manual overrides in the final Excel output.


12. TROUBLESHOOTING
--------------------------------------------------------------------------------

   Extension not loading:
     - Make sure montoya-api.jar is in the lib/ folder.
     - Make sure you built with: mvn clean package -q
     - Check Burp's Errors tab for stack traces.

   Tab names not auto-capturing:
     - Auto-capture only works for Repeater requests.
     - Scanner/Intruder requests are not captured.
     - Use right-click "Send to Checklist Generator" as a backup.

   Wrong TO mapping:
     - Add TO.XX explicitly in your tab name for 95% confidence.
     - Add CWE-XXX for 92% confidence.
     - Or just edit the TO Code cell in the results table.

   API mode failing:
     - Check your API key is entered correctly.
     - Check the API URL is correct.
     - Extension falls back to Local Engine automatically on API failure.

   Excel not opening after export:
     - Check the output path is valid and not locked by another program.
     - Try changing the output path to a different location.
